Features

14 modules, 85+ tools, 27 MCP endpoints, 4-tier memory, OSINT enrichment, Claude Agent SDK orchestration, and governance-first architecture.

Claude Agent SDK orchestration

Built on the Claude Agent SDK with per-phase model selection — Opus for orchestration, Sonnet for assessment and exploitation, Haiku for recon and reporting. Parallel sub-agent delegation with anti-stop nudges and adaptive continuation.

85+ tools in Kali containers

86 security tools across 10 categories (recon, web discovery, injection, exploitation, and more) execute inside sandboxed Kali containers with dynamic tool discovery on startup and automatic OPSEC defaults.

27 MCP agent tools

Shell execution, HTTP requests, memory operations, credential management, 6 security scanner wrappers (nmap, nikto, sqlmap, nuclei, ffuf, multipart), payload mutation, attack graph edges, sub-agent dispatch, and screenshot capture.

4-tier cross-run memory

Working memory (current run), short-term (per-target), long-term (global playbooks), and muscle memory (proven payloads with NEW → LEARNING → PROVEN promotion). Docker volume persistence across runs with target fingerprinting.

OSINT intelligence

Pre-assessment enrichment from 5 sources — Shodan (ports and services), Censys (certificates), VirusTotal (reputation), NVD (CVE search), and BuiltWith (technology profiling). Selective API key activation.

Deterministic recon sweep

Pre-LLM reconnaissance with ffuf and nuclei discovers endpoints and technologies before the agent session begins. Smart SPA size filtering eliminates false positives from single-page applications.

Immutable governance

5-stage command validation — allowlist, pattern blocking, scope check, risk classification, and default injection — with 5 scope profiles. Frozen dataclasses, opaque credential references, and output sanitization with 14+ secret pattern redaction.

Multi-pass orchestrator

4-pass strategy — breadth discovery, gap analysis, exploitation, reporting — with recursive sub-agent delegation, adaptive time reallocation, stagnation detection, and coverage matrix tracking across vulnerability types.

Multi-provider LLM routing

Phase-aware 3-tier routing across 6 providers (Anthropic, OpenAI, Gemini, Bedrock, Ollama, LM Studio) plus a proxy layer. Per-model token pricing and cost budget enforcement.

Payload mutation engine

30 transforms across encoding, syntax, and evasion categories. Chainable pipeline for WAF bypass with vuln-type-aware techniques for SQLi, XSS, RCE, LFI, and SSTI. Proven payloads are promoted in muscle memory.

Attack graph + deduplication

Directed graphs of attack paths with Mermaid and JSON rendering, DFS path enumeration, and LLM prompt injection. Pairwise similarity scoring with greedy merge and contradiction detection for clean reports.

Plugin system + evidence capture

5 TOML + Markdown plugins (general, CTF, bug bounty, code security, threat emulation) with dynamic tool loading. Headless Chromium screenshots, artifact collection, and structured JSON + Markdown reporting.

How it works

CLI-driven lifecycle: configure, enrich, execute, report — all within governance boundaries.

  1. 01

    Configure scope & target

    Select a scope profile (vuln_lab, bug_bounty, ctf, recon_only, full_auto), set your target, choose LLM providers per tier, and optionally enable OSINT API keys for pre-assessment intelligence.

  2. 02

    Recon sweep & OSINT enrichment

    Before the LLM session begins, a deterministic recon sweep runs ffuf and nuclei to discover endpoints and technologies. OSINT sources (Shodan, Censys, VirusTotal, NVD, BuiltWith) enrich the target profile. 4-tier memory loads proven payloads and prior findings.

  3. 03

    Agent executes autonomously

    The Claude Agent SDK orchestrator runs a 4-pass strategy — breadth discovery, gap analysis, exploitation, reporting — delegating to sub-agents with per-phase model selection. Tools execute in sandboxed Kali containers, payloads mutate to bypass WAFs, attack graphs grow as findings connect, and coverage matrices track tested vulnerability types.

  4. 04

    Review structured reports

    JSON and Markdown reports with confidence-scored, deduplicated findings, attack graph visualization, reproduction steps, screenshot evidence, and cost tracking. Proven payloads are promoted in muscle memory for future runs.
