Features

Opus orchestrator with sub-agent delegation, 86 tools in Kali containers, 37 MCP endpoints, 68 registered tools, smart recon sweep with fallback chains, 4-tier memory, OSINT enrichment, and governance-first architecture.

Opus orchestrator + sub-agents

Built on the Claude Agent SDK with an Opus orchestrator dispatching specialized sub-agents — enumerator (Haiku), assessor (Sonnet), exploiter (Sonnet), reporter (Haiku). Parallel delegation with anti-stop nudges, severity escalation, and adaptive continuation across coverage gaps.

86 tools in Kali containers

86 security tools across 10 categories execute inside sandboxed Kali containers built on kalilinux/kali-rolling. Dynamic tool discovery on startup with automatic OPSEC defaults (rate limits, timeouts, random user-agents) injected per tool.

37 MCP agent tools

Shell execution, HTTP requests, memory operations, credential management, 6 security scanner wrappers (nmap, nikto, sqlmap, nuclei, ffuf, multipart), API discovery (OpenAPI, GraphQL introspection, CORS check, schema fuzz), payload mutation, attack graph edges, sub-agent dispatch, and screenshot capture.

Smart recon + fallback chains

Pre-LLM deterministic reconnaissance — baseline HTTP probe, ffuf endpoint discovery with SPA size filtering, nuclei CVE scanning, and automatic fallback chains when primary tools fail (nuclei → nikto, ffuf → parse_js, browser_probe → parse_html). 5 output parsers normalize results across tools.

4-tier cross-run memory

Working memory (current run), short-term (per-target), long-term (global playbooks), and muscle memory (proven payloads with NEW → LEARNING → PROVEN promotion). Docker volume persistence across runs with target fingerprinting and post-run extraction.

OSINT intelligence

Pre-assessment enrichment from 5 sources — Shodan (ports and services), Censys (certificates), VirusTotal (reputation), NVD (CVE search), and BuiltWith (technology profiling). Selective API key activation.

Immutable governance

5-stage command validation — allowlist, pattern blocking, scope check, risk classification, and default injection — with 5 scope profiles. Frozen dataclasses, opaque credential references, and output sanitization with 14+ secret pattern redaction.

Payload mutation engine

30 transforms across encoding, syntax, and evasion categories. Chainable pipeline for WAF bypass with vuln-type-aware techniques for SQLi, XSS, RCE, LFI, and SSTI. Proven payloads are promoted in muscle memory.

Multi-provider LLM routing

Phase-aware 3-tier routing across 6 providers (Anthropic, OpenAI, Gemini, Bedrock, Ollama, LM Studio) plus a proxy layer. Per-model token pricing and cost budget enforcement.

Attack graph + deduplication

Directed graphs of attack paths with Mermaid and JSON rendering, DFS path enumeration, and LLM prompt injection. Pairwise similarity scoring with greedy merge and contradiction detection for clean reports.

Plugin system + evidence capture

5 TOML + Markdown plugins (general, CTF, bug bounty, code security, threat emulation) with dynamic tool loading. Headless Chromium screenshots, artifact collection, and structured JSON + Markdown reporting.

Confidence scoring + dedup

Bayesian confidence scoring with configurable weights. Finding deduplication via pairwise similarity with greedy merge and contradiction detection. Type normalization with 21 canonical vulnerability categories.

How it works

CLI-driven lifecycle: configure, recon, execute, report — all within governance boundaries.

  1. 01

    Configure scope & target

    Select a scope profile (vuln_lab, bug_bounty, ctf, recon_only, full_auto), set your target, choose LLM providers per tier, and optionally enable OSINT API keys for pre-assessment intelligence.

  2. 02

    Recon sweep & OSINT enrichment

    Before the LLM session begins, a smart recon sweep runs ffuf (with SPA size filtering) and nuclei to discover endpoints and CVEs — with fallback tool chains (nuclei fails → nikto, ffuf fails → parse_js) for resilience. OSINT sources (Shodan, Censys, VirusTotal, NVD, BuiltWith) enrich the target profile. 4-tier memory loads proven payloads and prior findings.

  3. 03

    Opus orchestrator executes

    The Opus orchestrator dispatches specialized sub-agents — enumerator (Haiku), assessor (Sonnet), exploiter (Sonnet) — for parallel vulnerability coverage. Tools execute in sandboxed Kali containers, payloads mutate to bypass WAFs, attack graphs grow as findings chain into critical escalations, and coverage matrices track all 21 canonical vulnerability types.

  4. 04

    Review structured reports

    JSON and Markdown reports with confidence-scored, deduplicated findings, attack graph visualization, reproduction steps, screenshot evidence, and cost tracking. Proven payloads are promoted in muscle memory for future runs.
