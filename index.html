Capabilities

Sandboxed execution, immutable scope, governance at every layer.

Orchestrator + sub-agents

An Opus orchestrator dispatches specialized sub-agents (enumerator, assessor, exploiter, reporter) across 6 LLM providers. Auto-hooks fire reactively on findings — no LLM call needed for JWT decode, JS parsing, API discovery, or CORS checks.

Pre-finding flow engine

Deterministic register → login → discover → exploit chains run inline before vuln type is even confirmed. Three triggers — recon conditions, 401/403 heuristics, and LLM-requested. Credential rotation across 6 sets, CSRF extraction, and registration fallback. Unlocks vulnerabilities hidden behind auth.

Multi-step attack chaining

Five deterministic kill chains restructured as phased ATT&CK paths: SSTI → RCE, SSRF → localhost pivot, SQLi → creds → auth bypass, file upload → webshell, IDOR → enumeration → privesc. Adaptive thinking on the orchestrator; chain phase tracking injected into every continuation prompt.

72 tools in Kali sandbox

Nmap, sqlmap, nuclei, ffuf, Burp Suite Pro, and 67 more — all inside containers with OPSEC defaults. 30 payload mutation transforms for WAF/filter evasion. Findings map to ATT&CK and OWASP taxonomies automatically.

Governance + coordination

Immutable scope, 5-stage command validation, credential isolation. A git-backed progress repo tracks every tested (endpoint, method, vuln_type, technique) intent — blocking duplicates at 3 attempts, surfacing coverage gaps to orchestrator and sub-agents alike.

Learns across runs

Three memory tiers — working, tradecraft, muscle — promote payloads from NEW to LEARNING to PROVEN across runs. Deterministic scanners and JS bundle analysis run before the LLM session. Each run makes the next one faster and deeper.

How it works

  1. 01

    Define scope

    Set target, objective, and scope profile. The governance engine freezes scope — it cannot expand mid-operation.

  2. 02

    Recon + auth

    Fast scanners map the surface while deferred scanners (nuclei, Burp) run async. JS bundles are analyzed for hidden endpoints and secrets. The pre-finding flow engine handles register/login/CSRF inline so authenticated surface is reachable before the orchestrator starts.

  3. 03

    Test, chain, exploit

    Sub-agents test in parallel. Five deterministic kill chains fire on confirmed findings (SSTI→RCE, SSRF→pivot, SQLi→creds, file upload→shell, IDOR→privesc). A progress repo blocks duplicate intents and surfaces coverage gaps.

  4. 04

    Report + learn

    Deduplicated findings with ATT&CK/OWASP mapping, attack graphs, and evidence. Proven payloads promote to tradecraft memory for future runs.
