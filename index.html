Features

Autonomous assessment with real security tooling, explicit governance, multi-provider LLM routing, payload mutation, attack graph mapping, and intelligence-driven scoring.

Multi-provider LLM routing

Phase-aware 3-tier routing across 6 providers (Anthropic, OpenAI, Gemini, Bedrock, Ollama, LM Studio). Mix and match vendors per assessment phase.

Sandboxed Kali execution

50+ security tools (nmap, sqlmap, nuclei, hydra, and more) execute inside isolated Docker containers with automatic OPSEC defaults.

Immutable governance

4-stage command validation pipeline with scope locking, pattern blocking, risk classification, and 5 built-in scope profiles.

OODA agent loop

Observe-Orient-Decide-Act cycle with vector memory, credential vault with opaque references, and structured JSON + Markdown reporting.

Cross-operation memory

Recall findings from previous operations with vector-similarity search. Pluggable persistence backends let the agent build on prior assessments.

Confidence scoring

Evidence-weighted scoring with tool-source weights, corroboration bonuses, and exploitation-proof multipliers for high-signal findings.

Payload mutation engine

Registry-based transform pipeline with ~35 mutations across encoding, syntax, and evasion categories. Chainable transforms for WAF bypass with vuln-type-aware techniques for SQLi, XSS, RCE, LFI, and SSTI.

False positive reduction

Pairwise finding similarity scoring with greedy merge algorithm and contradiction detection. Automatically deduplicates near-duplicate findings for cleaner reports.

Attack graph construction

Builds directed graphs of attack paths as the agent discovers relationships between findings. Mermaid flowchart and JSON rendering with DFS path enumeration, injected into LLM prompts and appended to final reports.

Screenshot capture

Headless Chromium via Playwright captures visual evidence of vulnerabilities and attaches screenshots directly to finding reports.

Plugin system

TOML + Markdown-based plugins with dynamic tool loading. Ship assessment modules for general, CTF, bug bounty, code security, and threat emulation.

How it works

CLI-driven lifecycle: configure, execute, report — all within governance boundaries.

  1. 01

    Configure scope & target

    Select a scope profile (vuln_lab, bug_bounty, ctf, recon_only, full_auto), set your target, and choose LLM providers per tier.

  2. 02

    Agent executes autonomously

    The OODA loop plans approach, recalls prior findings, dispatches tools in sandboxed containers, mutates payloads to bypass WAFs, builds attack graphs linking related findings, captures screenshot evidence, and pivots based on results — all validated through the 4-stage command pipeline.

  3. 03

    Review structured reports

    JSON and Markdown reports with confidence-scored, deduplicated findings, attack graph visualization, reproduction steps, screenshot evidence, and reasoning. Secrets are never exposed — credentials use opaque references throughout.
