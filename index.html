Capabilities

Sandboxed execution, immutable scope, governance at every layer.

Orchestrator + sub-agents

Opus orchestrator dispatches specialized sub-agents (enumerator, assessor, exploiter, reporter) across 6 LLM providers. Auto-hooks fire reactively — JWT decode, JS parsing, API discovery, CORS checks — without LLM involvement.

72 tools in Kali sandbox

Nmap, sqlmap, nuclei, ffuf, Burp Suite Pro, and 67 more — all inside containers with OPSEC defaults injected automatically. Condition-based dispatch fires reactive tools on findings. Fallback chains handle tool failures.

Governance + progress tracking

Immutable scope, 5-stage command validation, credential isolation. Git-backed progress repo tracks every tested endpoint/vuln/technique tuple — blocking duplicates before they consume budget and surfacing untested attack surface.

Recon pipeline + memory

Deterministic scanners and JS bundle analysis run before the LLM session. Deferred scanners deliver results async. Three-tier cross-run memory promotes proven payloads so the agent improves over time.

How it works

  1. 01

    Define scope

    Set target, scope profile, and providers. The governance engine freezes scope — it cannot expand mid-operation.

  2. 02

    Recon

    Deterministic scanners run before the LLM — endpoint discovery, JS bundle analysis, OSINT enrichment. Fast scanners block; deferred scanners run async and deliver results via callback.

  3. 03

    Agent execution

    Sub-agents test in parallel. Auto-hooks fire reactive tools on findings. Git-backed progress repo tracks every tested tuple, blocks duplicates, and surfaces untested attack surface via read_progress.

  4. 04

    Output

    JSON and Markdown reports with deduplicated findings, attack graph, and evidence. Proven payloads promote to muscle memory for future runs.
