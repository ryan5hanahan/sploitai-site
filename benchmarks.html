Methodology

XBEN suite

104 CTF-style challenges across 3 difficulty levels. Each is a containerized vulnerable app with a known flag. Pass requires capturing the flag and correctly identifying the vulnerability type.

Adjusted rate

Build failures (stale base images) are excluded from the denominator. 53% of the suite currently fails to build. Adjusted rate = passes / runnable challenges.

Execution model

Fully autonomous — no human intervention after launch. Agent runs inside per-scan isolated Kali containers with 2-layer governance. Anti-hallucination pipeline validates every finding.

Use responsibly. Always test only against systems you own or have explicit permission to assess.
