Methodology

Benchmark suite

XBEN — 104 CTF-style challenges across 2 difficulty levels. Each challenge is a containerized vulnerable application with a known flag.

Pass criteria

A challenge passes when the agent captures the flag AND correctly identifies the vulnerability type. Flag-only or type-only does not count.

Adjusted rate

Build failures (broken challenge containers) are excluded from the denominator. Raw rate counts all 104; adjusted rate counts only runnable challenges.

Execution model

Fully autonomous — no human intervention after launch. Agent runs inside sandboxed Kali containers with immutable scope policies.

Juice Shop runs

Separate from XBEN. Real-world target (OWASP Juice Shop) measuring findings count, vuln type diversity, step efficiency, and duration.

Progress repo

Git-backed intent tracking records every tested endpoint/vuln/technique tuple. Blocks duplicate work and surfaces untested attack surface.
